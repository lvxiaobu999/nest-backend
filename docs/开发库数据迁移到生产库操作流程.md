# 开发库数据迁移到生产库操作流程

本文档用于把开发环境 PostgreSQL 数据迁移到阿里云生产环境 PostgreSQL。

当前项目配置来自：

- `.env.development`
- `.env.production`
- `docker-compose.yml`

## 一、当前环境信息

### 开发环境

```text
COMPOSE_PROJECT_NAME=nest-frontend-dev
PostgreSQL 容器名=nest-frontend-dev-postgres
PostgreSQL 宿主机端口=5432
PostgreSQL 容器端口=5432
数据库用户=postgres
数据库名=nest_prisma_dev
```

### 生产环境

```text
COMPOSE_PROJECT_NAME=nest-frontend-prod
PostgreSQL 容器名=nest-frontend-prod-postgres
PostgreSQL 宿主机端口=5433
PostgreSQL 容器端口=5432
数据库用户=postgres
数据库名=nest_prisma_prod
```

## 二、迁移前确认

生产环境如果已经有真实用户数据，不要直接覆盖。本文档默认你的目标是：

```text
用开发库 nest_prisma_dev 的数据完整覆盖生产库 nest_prisma_prod
```

先在生产服务器确认容器正在运行：

```bash
docker ps
```

应该能看到：

```text
nest-frontend-prod-postgres
nest-frontend-prod-redis
```

在开发机器确认开发库容器正在运行：

```bash
docker ps
```

应该能看到：

```text
nest-frontend-dev-postgres
nest-frontend-dev-redis
```

## 三、先备份生产库

在阿里云生产服务器执行：

```bash
cd /你的生产项目目录

docker exec -t nest-frontend-prod-postgres pg_dump \
  -U postgres \
  -d nest_prisma_prod \
  -Fc \
  > prod-backup-before-dev-import.dump
```

确认备份文件存在：

```bash
ls -lh prod-backup-before-dev-import.dump
```

这一步很重要。即使生产库现在看起来是空的，也建议先备份一次。

## 四、从开发库导出数据

在开发机器的项目目录执行：

```bash
cd /你的开发项目目录

docker exec -t nest-frontend-dev-postgres pg_dump \
  -U postgres \
  -d nest_prisma_dev \
  -Fc \
  > dev-to-prod.dump
```

确认导出文件存在：

```bash
ls -lh dev-to-prod.dump
```

如果你的开发机器是 Windows PowerShell，不要直接复制 Bash 里的 `\` 换行命令。PowerShell 的换行符是反引号 `` ` ``，而且 `pg_dump -Fc` 是二进制格式，不建议用 PowerShell 的 `>` 直接重定向。

推荐在 Windows PowerShell 中执行下面这组命令：

```powershell
cd D:\study\nest\nest-fontend

docker exec nest-frontend-dev-postgres pg_dump -U postgres -d nest_prisma_dev -Fc -f /tmp/dev-to-prod.dump

docker cp nest-frontend-dev-postgres:/tmp/dev-to-prod.dump .\dev-to-prod.dump

docker exec nest-frontend-dev-postgres rm -f /tmp/dev-to-prod.dump

Get-Item .\dev-to-prod.dump
```

如果 `Get-Item` 能看到 `dev-to-prod.dump` 且文件大小不是 0，说明导出成功。

## 五、上传导出文件到生产服务器

在开发机器执行：

```bash
scp dev-to-prod.dump root@你的ECS公网IP:/你的生产项目目录/dev-to-prod.dump
```

如果你的生产项目目录是 `/var/www/nest-backend`，则执行：

```bash
scp dev-to-prod.dump root@你的ECS公网IP:/var/www/nest-backend/dev-to-prod.dump
```

如果你的生产项目目录是 `/www/nest-backend`，则执行：

```bash
scp dev-to-prod.dump root@你的ECS公网IP:/www/nest-backend/dev-to-prod.dump
```

## 六、停止生产应用

导入前建议先停止 Nest 应用，避免导入时还有连接占用数据库。

在生产服务器执行：

```bash
pm2 list
```

如果应用名是 `nest-backend-api`：

```bash
pm2 stop nest-backend-api
```

如果你的 PM2 应用名不同，用 `pm2 list` 里显示的真实名称替换。

## 七、清空并重建生产库

在生产服务器执行：

```bash
cd /你的生产项目目录

docker exec -i nest-frontend-prod-postgres dropdb \
  -U postgres \
  nest_prisma_prod

docker exec -i nest-frontend-prod-postgres createdb \
  -U postgres \
  nest_prisma_prod
```

如果 `dropdb` 提示还有连接占用，可以执行：

```bash
docker exec -i nest-frontend-prod-postgres psql \
  -U postgres \
  -d postgres \
  -c "SELECT pg_terminate_backend(pid) FROM pg_stat_activity WHERE datname = 'nest_prisma_prod' AND pid <> pg_backend_pid();"
```

然后重新执行：

```bash
docker exec -i nest-frontend-prod-postgres dropdb \
  -U postgres \
  nest_prisma_prod

docker exec -i nest-frontend-prod-postgres createdb \
  -U postgres \
  nest_prisma_prod
```

## 八、导入开发库数据到生产库

在生产服务器执行：

```bash
cd /你的生产项目目录

docker exec -i nest-frontend-prod-postgres pg_restore \
  -U postgres \
  -d nest_prisma_prod \
  --no-owner \
  --no-privileges \
  < dev-to-prod.dump
```

如果没有报错，说明导入完成。

## 九、执行 Prisma 生产迁移

导入完成后，在生产服务器执行一次 Prisma 生产迁移，确保数据库结构和当前代码一致：

```bash
pnpm run prisma:migrate:deploy
```

如果你只是把同一份代码对应的开发库完整迁移到生产库，通常不会有额外变更；但生产环境执行这一步更稳。

## 十、重启生产应用

在生产服务器执行：

```bash
pm2 restart nest-backend-api
```

如果应用之前是停止状态，也可以执行：

```bash
pm2 start nest-backend-api
```

如果你的 PM2 应用名不同，用 `pm2 list` 里显示的真实名称替换。

## 十一、验证生产数据

在生产服务器进入 PostgreSQL：

```bash
docker exec -it nest-frontend-prod-postgres psql \
  -U postgres \
  -d nest_prisma_prod
```

进入后查看表：

```sql
\dt
```

查看某张表数据量，例如：

```sql
SELECT COUNT(*) FROM "User";
```

退出：

```sql
\q
```

也可以用 DBeaver 连接生产库验证：

```text
Host=你的ECS公网IP
Port=5433
Database=nest_prisma_prod
Username=postgres
Password=使用 .env.production 中的 POSTGRES_PASSWORD
```

更推荐使用 SSH 隧道连接，不直接暴露数据库端口：

```bash
ssh -L 15433:127.0.0.1:5433 root@你的ECS公网IP
```

DBeaver 中填写：

```text
Host=127.0.0.1
Port=15433
Database=nest_prisma_prod
Username=postgres
Password=使用 .env.production 中的 POSTGRES_PASSWORD
```

## 十二、如果需要回滚

如果导入后发现生产环境数据不对，可以用第三步生成的备份恢复。

先停止应用：

```bash
pm2 stop nest-backend-api
```

重建生产库：

```bash
docker exec -i nest-frontend-prod-postgres dropdb \
  -U postgres \
  nest_prisma_prod

docker exec -i nest-frontend-prod-postgres createdb \
  -U postgres \
  nest_prisma_prod
```

恢复备份：

```bash
docker exec -i nest-frontend-prod-postgres pg_restore \
  -U postgres \
  -d nest_prisma_prod \
  --no-owner \
  --no-privileges \
  < prod-backup-before-dev-import.dump
```

重启应用：

```bash
pm2 restart nest-backend-api
```

## 十三、Redis 是否需要迁移

当前 `docker-compose.yml` 中 Redis 使用：

```text
container_name=${COMPOSE_PROJECT_NAME}-redis
command=redis-server --appendonly yes
```

也就是：

```text
开发 Redis 容器=nest-frontend-dev-redis
生产 Redis 容器=nest-frontend-prod-redis
```

一般不建议把开发环境 Redis 数据迁移到生产环境。Redis 通常存的是缓存、验证码、临时 token、限流计数等临时数据，迁移到生产环境可能造成登录态、验证码、缓存污染。

除非你确认 Redis 里存的是必须保留的业务数据，否则只迁移 PostgreSQL。

## 十四、完整命令速查

开发机器导出：

```bash
docker exec -t nest-frontend-dev-postgres pg_dump -U postgres -d nest_prisma_dev -Fc > dev-to-prod.dump
scp dev-to-prod.dump root@你的ECS公网IP:/你的生产项目目录/dev-to-prod.dump
```

生产服务器备份并导入：

```bash
cd /你的生产项目目录

docker exec -t nest-frontend-prod-postgres pg_dump -U postgres -d nest_prisma_prod -Fc > prod-backup-before-dev-import.dump

pm2 stop nest-backend-api

docker exec -i nest-frontend-prod-postgres psql -U postgres -d postgres -c "SELECT pg_terminate_backend(pid) FROM pg_stat_activity WHERE datname = 'nest_prisma_prod' AND pid <> pg_backend_pid();"

docker exec -i nest-frontend-prod-postgres dropdb -U postgres nest_prisma_prod
docker exec -i nest-frontend-prod-postgres createdb -U postgres nest_prisma_prod

docker exec -i nest-frontend-prod-postgres pg_restore -U postgres -d nest_prisma_prod --no-owner --no-privileges < dev-to-prod.dump

pnpm run prisma:migrate:deploy
pm2 restart nest-backend-api
```
